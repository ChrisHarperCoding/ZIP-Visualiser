# Security Policy

## Supported Versions

We / I take security seriously and strive to provide timely security updates for our software. The following versions are currently being supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 5.1.x   | :white_check_mark: |
| 5.0.x   | :x:                |
| 4.0.x   | :white_check_mark: |
| < 4.0   | :x:                |

Please note that if a version is no longer supported, it means that we will not be providing security updates for that version. 

We strongly encourage users to upgrade to a supported version to ensure the security of their software.


## Reporting a Vulnerability

If you believe you have found a security vulnerability in our software, please report it to us as soon as possible. You can do so by raising an issue in this repo.

Once we receive your report, we will investigate the issue as soon as possible. We may ask for additional information or clarification during this time.

If the vulnerability is accepted, we will work to provide a fix as soon as possible. 
We will keep you updated on our progress as much as possible and provide an estimated timeline for when the fix will be released, if one is available.

If the vulnerability is declined, we will provide an explanation for our decision and any steps you can take to mitigate the issue.

We appreciate your help in keeping our software secure.
